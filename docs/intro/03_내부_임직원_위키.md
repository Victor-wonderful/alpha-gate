# Alpha Gate — 내부 임직원 위키 (초안)

> 형식: 내부 온보딩/레퍼런스 위키. 확정 후 Notion 또는 사내 문서로 이관.
> 대상: 신규 입사자 + 전 임직원. "회사·제품·기술·운영"을 한 번에 파악.

## 0. 이 문서는 무엇인가

Alpha Gate를 처음 접하는 임직원이 **제품의 정체성·작동 원리·기술 구조·운영 방식·로드맵**을 한 번에 이해하도록 정리한 단일 출처. 코드 세부는 `CLAUDE.md`와 `src/` 주석을 참조.

## 1. 회사·제품 정체성

### 한 줄 정의
**매매 전 의사결정 체크 + AI 분석.** 암호화폐 트레이더가 진입 버튼을 누르기 전에 "이 거래를 해도 되는가"를 검증하게 만드는 도구.

### 핵심 메시지 (전 직원 통일)
- "무엇을 살까"가 아니라 **"이 거래를 해도 되는가"**에 답한다.
- 트레이더가 잃는 이유는 분석 부족이 아니라 **결정의 일관성 붕괴**.
- 한 번의 D급 거래가 한 달 수익을 지운다 → 진입 전에 막는다.

### 브랜드
- 이름: **Alpha Gate** (구 TradeGate에서 변경)
- 로고: 글자 없는 두 겹 셰브론(순수 기하)
- 연계 자산: **Victor Alpha 블로그** — 통합 브랜드, 사이드바 하단에서 외부 링크 연결
- 톤: 평범한 한국어 우선, 전문 용어 최소화, 심플 우선 + "전문가 정보 보기" 펼치기

## 2. 제품 전체 지도 (사용자 여정)

### 핵심 4단계 루프
1. **AI 분석** (`/app/analyze`) — 진입 전 시장 분석, 시나리오 1~3개 제시
2. **주문 검토** (`/app/trade`) — 진입가/손절/목표 입력 → A·B·C·D 등급
3. **내 거래 / 저널** (`/app/journal`) — 진입 평가 보존 + 청산 결과 + AI 복기
4. **성과 분석** (`/app/dashboard`) — 등급별 평균 R·승률·실수 태그 통계

### 부가 기능 (라우트 기준)
| 라우트 | 기능 |
|---|---|
| `/app` | 홈 대시보드 (통계 + 시세 + 최근 거래) |
| `/app/analyze` | AI 분석 + 후보 레이더("지금 볼 만한 코인") + 백테스트 모드 |
| `/app/trade` | 주문 검토 (등급·사이징·자금관리·시장 컨텍스트) |
| `/app/virtual-trade` | 트레이딩 터미널 (가상 거래, 선물/현물) |
| `/app/arbitrage` | 김치 프리미엄 차익거래 (인벤토리 리밸런싱 시뮬) |
| `/app/game` | 다음 캔들 예측 게임 |
| `/app/rankings` | 게임/성과 랭킹 |
| `/app/wallet`, `/app/deposit`, `/app/credits` | 가상지갑(vUSDT) + 입금 + AI 크레딧 구매 |

## 3. AI 분석 파이프라인 (핵심 도메인 — 반드시 이해)

단일 LLM 호출이 아니라 **3단계 + 저장** 구조. 책임 분리가 핵심.

### Stage 1 — Market Data (결정론적 코드)
`src/lib/analysis/analyze.ts: buildSnapshot()`
- Binance Futures: 멀티 TF 캔들(3개)·호가창·체결(aggTrades)·펀딩비·OI·24h 티커
- CoinGecko: BTC 도미넌스
- 모두 병렬 fetch
- 자체 계산: 스윙 포인트, FVG, 오더블록, 유동성 존, 볼륨 프로파일(POC/VAH/VAL), 오더플로우

### Stage 2 — Strategy Agent (경량 LLM)
`src/lib/analysis/strategy.ts: classifyStrategy()`
- 스냅샷을 보고 **5개 전략 중 1개** 선택: 추세눌림 / 돌파 / 박스 / 반전 / 관망
- 출력: `{ primary, direction, confidence, reasoning, rejected }`
- 평범한 한국어 응답 강제, 시스템 프롬프트 캐싱

### Stage 3 — Scenario Generator (메인 LLM)
`src/lib/analysis/synthesize.ts: synthesizeAnalysis()`
- Stage 2 전략 안으로 시나리오를 **제약**
- 출력: `{ summary, structure, keyLevels, flow, scenarios[], actionNow, warnings }`
- **스타일별 표준 손절/목표/RR 강제** (벗어나면 시나리오 거부)
- 차트 이미지 첨부 시 보조 컨텍스트로만 사용 — **데이터가 ground truth**

### Stage 4 — Persist (코드)
`src/lib/analysis/persist.ts: saveAnalysis()` — 자동 Supabase 저장, `?load=<id>`로 복원

> **왜 이렇게 했나**: LLM에 가격을 직접 묻지 않는다 → 환각 차단 + 검증 가능. 이게 우리의 기술 해자이자 마케팅 메시지("객관 데이터 기반").

## 4. 도메인 규칙 (제품 일관성의 근간)

### 트레이딩 스타일 4종 (`style.ts`)
| ID | 라벨 | HTF/MTF/LTF |
|---|---|---|
| scalp | 스캘핑 (수분~수시간) | 1H/15M/5M |
| day | 데이 (수시간~하루) | 4H/1H/15M |
| swing | 스윙 (며칠~수주) ⭐기본 | 1D/4H/1H |
| position | 포지션 (수주~수개월) | 1D/4H/1H |

### 표준 손절/목표/RR (`standards.ts`, 왕복비용 0.12% 가정)
| 스타일 | 손절폭 | 목표폭 | 최소 R:R |
|---|---|---|---|
| 스캘핑 | 0.3~0.7% | 0.7~1.5% | 2+ |
| 데이 | 0.7~1.5% | 1.5~3% | 1.5+ |
| 스윙 | 2~5% | 5~15% | 2+ |
| 포지션 | 5~15% | 15~50% | 3+ |

### 등급 시스템 (`grading.ts`)
- 점수 합계 → ≥8 **A**(좋음) / 5~7 **B**(조건부) / 2~4 **C**(비추천) / ≤1 **D**(강한 자제)
- 가산: 손익비 ≥3 +3 / ≥2 +2 / ≥1.5 +1, 손절 구조 명확 +2, BTC 정렬 +1, 거래량 +1 등
- 감산: 손익비 무효 −2, BTC 충돌 −2, 박스 중간 진입 −2, 연속 손실/일일한도 −2, 동일코인 중복·과노출 등
- **D 원인 분기**: 계좌발(일일한도/과노출)="오늘은 보류"(firm) vs 셋업발="고위험·10% 축소(막지 않음)". 실거래 하드차단(`live-guards.ts`)은 별개로 유지.

### 포지션 사이징 (`sizing.ts`) — 리스크 % 기반
- `maxLoss = 계좌 × 허용손실%`, `quantity = maxLoss / |entry-stop|`
- 용어 통일: 잃을 한도 / 노출 금액 / 매수 수량 / 필요 마진 / 실효 손익비(수수료 0.12% 차감 후 R)

## 6. 데이터 모델 핵심 (Supabase)

- `profiles` — 사용자 기본 설정 (스타일·리스크·레버리지·disabled·role)
- `trades` — 진입 입력 + 진입 평가(pre_grade/score/rr) + 결과(exit/result_r/mistake_tags) + AI 복기. mode(live/backtest), market_type(futures/spot)
- `analyses` — 분석 결과 영구 저장 (snapshot/strategy/report JSONB)
- `paper_wallets` — 가상지갑(vUSDT) 잔액 + AI 크레딧
- `scenario_outcomes` — 시나리오별 가격 추적(워치 알림)
- `radar_candidates` — 후보 레이더 스캔 결과
- `arbitrage_positions` / `arbitrage_cycles` / `kimchi_history` — 김프 차익거래
- `notification_channels` / `notification_log` — 알림
- `admin_audit_logs` — 어드민 감사 로그
- **모든 사용자 테이블 RLS: 본인 행만 접근.** service_role은 서버/cron에서만.

> 마이그레이션: `supabase/migrations/0001~0034`. dev=prod 동일 Supabase 프로젝트 (마이그레이션 1회로 양쪽 반영).

- **구독 SaaS**: Free ₩0(월5회) / Standard ₩15,000(월100회) / Pro ₩95,000(월500회) / Premium ₩295,000(무제한).

### ⚠️ 알려진 운영 함정 (반드시 숙지)
1. **Vercel 자동배포 끊김** (2026-06-02~): GitHub App이 repo 접근 권한 상실 → main push해도 자동배포 안 됨. **우회: `cd D:/web01; vercel --prod` (`.vercelignore` 필수)**. 대시보드 Settings→Git 재연결 필요.
2. **ANTHROPIC_API_KEY 빈 값**: Claude Code 셸이 키를 빈 문자열로 export → dev 서버는 별도 PowerShell 창에서 띄워야 AI 분석 작동.
3. **dev 서버 켠 채 `git checkout` 금지**: Windows 파일 락으로 실패. dev 먼저 끄고 브랜치 조작.
4. **lightweight-charts "Object is disposed"**: 글로벌 핸들러 + ChartErrorBoundary로 차단됨.

### 환경변수 (`.env.local`)

## 10. 보안 / 컴플라이언스 (비밀)

- RLS 본인 행만. service_role은 서버 전용.
- ⚠️ **보류 중 리스크**: Supabase Secret Key 폐기+재발급 미완 (의도적 보류, 개발 우선). 키 노출 위험 인지 상태.
