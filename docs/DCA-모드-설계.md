> **상태:** 설계 확정 대기 (v0.1 draft, 2026-07-13)
> **구현:** 미착수 — 로드맵 등재
> **연관 결정:** 포지션 스타일 선물 UI 숨김 (같은 날, 이 문서가 그 대체 그릇)

## 1. 배경 — 왜 만드나

### 1.1 포지션 스타일의 구조적 실패 (데이터)
- `scenario_outcomes` 포지션 스타일: **결정 13건 전패** (승률 0%, 평균 −3.4R, 트리거율 76%).
- 원인은 우연이 아니라 구조: 선물 시나리오 틀(진입/손절/목표)에 수개월 보유를 넣으면, 크립토 변동성에서 5~15% 손절은 보유 기간 중 거의 반드시 스윕됨.
- 무기한 선물로 수개월 보유 자체가 부적합: 펀딩비(평균 +0.01%/8h ≈ 월 0.9%)가 누적 출혈.

### 1.2 결론 (Victor 확정)
> 장기 투자의 맞는 그릇 = **현물 + 레버리지 없음 + 분할 매수(DCA)**.
> 포지션 스타일은 선물 UI에서 숨기고, 장기 수요는 이 DCA 모드로 이관한다.

### 1.3 VECTA 철학과의 연결
핵심 메시지 "이 거래를 해도 되는가"의 자연스러운 확장 = **"이 매집을 해도 되는가."**
- 선물 게이트가 막는 것: 충동 진입, 추격, 복수매매.
- DCA 게이트가 막는 것: **고점 FOMO 몰빵, 잡코인 물타기, 무계획 매수.**
- 같은 제품 철학, 다른 의사결정 모델. 주 1회 사용 습관 → retention 자산.

## 2. 의사결정 모델 — 선물 시나리오와 무엇이 다른가

| 축 | 선물 시나리오 (기존) | 현물 적립 (DCA 모드) |
|---|---|---|
| 핵심 질문 | 이 거래를 해도 되는가 | 이 **매집**을 해도 되는가 |
| 방어 수단 | 손절가 (R 기반) | ❌ 손절 없음 — **자산 게이트 + 배분 한도 + 분할** |
| 진입 | 단일 진입가 + 트리거 | **가격 사다리 / 주기 분할** (여러 번) |
| 사이징 | 리스크 % → 수량 | **포트폴리오 배분 %** (코인당 상한) |
| 타이밍 | 캔들 마감·세션 | **밸류 존** (지금 싼 구간인가) |
| 청산 | 목표가 / 만료 | 사이클 단위 (명시적 목표 없음, 상한 도달 시 매집 중지) |
| 성과 척도 | R | **평단 대비 / 미배분 예산 소진율 / 규율 준수율** |

## 3. 4개 게이트 (등급 로직의 DCA 버전)

모두 **코드 결정론** — AI 독립 원칙 유지 (기존 marketAssessment 패턴).

### G1. 자산 게이트 — "모아도 되는 자산인가" (통과 못 하면 차단)
DCA는 "물릴수록 더 사는" 전략이라 **죽지 않는 자산**에만 허용된다.
- 유니버스: **시총 상위 15 큐레이션 상수 재사용** (`MEGA_CAP_UNIVERSE`, radar.ts) — 이미 검증된 대장주 목록.
- 추가 조건(코드): 상장 연차 ≥ 2년(일봉 개수로 판정), 현물 시장 존재(api.binance.com), 스테이블 제외.
- 통과 실패 = **하드 차단** ("이 코인은 적립 대상이 아닙니다 — 죽을 수 있는 자산은 물타기 금지").

### G2. 밸류 존 — "지금 싼 구간인가" (점수)
기존 "3지표 다수결" 패턴(추세·지표 카드) 재사용. 일봉/주봉 기반 3지표:
1. **고점 대비 낙폭 백분위** — 현재 낙폭이 이 코인 역사에서 상위 몇 %인가 (깊을수록 +)
2. **200D MA 대비 위치** — 아래면 +, 크게 위면 −
3. **주봉 Volume Profile VAL 근접** — 장기 매물대 하단 근처면 + (기존 volume-profile.ts 재사용)

다수결 → `cheap / neutral / expensive` 3단 판정. expensive면 "지금은 모을 때가 아니다 — 대기".

### G3. 배분 한도 — "너무 몰빵 아닌가" (점수/경고)
- 프로필에 총 투자예산 + 코인당 배분 상한 설정 (기본: BTC 40% / ETH 25% / 기타 10%).
- 현재 보유(가상 현물 지갑) + 이번 매수 후 배분이 상한 초과 → **감점/차단**.
- 기존 money-management(노출 집계) 패턴의 현물 버전.

### G4. 스케줄 규율 — "계획대로 사는가" (점수)
- 플랜에 정의된 주기(예: 격주) 또는 가격 사다리(예: −5%마다) **밖의 매수 = 감점** ("이건 DCA가 아니라 충동 매수입니다").
- 직전 매수 후 최소 간격 미달 경고. — 선물의 연속손실 감점과 같은 역할.

**등급 산출:** G1 통과 전제 + G2/G3/G4 점수 합산 → 기존 A/B/C/D 등급 배지 재사용 (grading.ts에 `mode: "dca"` 분기 또는 별도 `dca-grading.ts`).

## 4. 적립 플랜 (핵심 오브젝트)

사용자가 만드는 것은 "주문"이 아니라 **플랜**:

```
DcaPlan {
  symbol            // BTCUSDT (현물)
  totalBudget       // 총 매집 예산 (USDT)
  tranches          // 분할 횟수 (예: 8회)
  mode              // "periodic" (격주/월) | "ladder" (가격 사다리 −X%마다) | "hybrid"
  schedule          // periodic: 주기. ladder: 기준가+간격%
  maxAllocationPct  // 포트폴리오 대비 상한
  status            // active / paused / completed
}
```

- 플랜 생성 시 **회당 금액 자동 계산** (totalBudget / tranches).
- 각 회차 실행은 기존 **가상 현물 매수**(order-actions spot 분기)로 기록 — `trades`에 `context_flags.dcaPlanId` 연결.
- 플랜 화면: 진행률(회차 소진), **평단 vs 현재가**, 배분 현황, 다음 예정 회차.

## 5. 알림 (기존 인프라 재사용)

- **밸류 존 진입**: G2가 cheap로 바뀌면 텔레그램 "지금 BTC가 밸류 존입니다 — 플랜 회차 실행 검토" (notify-dispatch 이벤트 추가, cron 일 1회면 충분 — DCA는 분 단위가 필요 없음).
- **주기 도래**: periodic 플랜의 다음 매수일 알림.
- 로드맵의 "워치리스트+트리거 알림"과 같은 기반 — 함께 구현하면 시너지.

## 6. UI 구성 (MVP)

- **신규 라우트 `/app/dca`** (사이드바 "적립" 메뉴) — 선물 분석과 명확히 분리(다른 의사결정 모델이므로 같은 화면에 섞지 않음).
  1. 상단: 자산 선택(G1 통과 자산만) + 밸류 존 현황 카드(3지표 다수결, 추세·지표 카드와 같은 골격)
  2. 중단: 내 플랜 목록 (진행률·평단·배분)
  3. 플랜 생성/실행 모달 → 등급 배지 + "이 매집을 해도 되는가" 요약 → [회차 실행] = 가상 현물 매수
- 거래일지: 기존 3탭에 DCA 필터 또는 `context_flags`로 구분 표시.

## 7. DB (신규 1테이블)

```sql
-- dca_plans: 위 DcaPlan 필드 + user_id + RLS(본인 행)
-- 회차 실행 기록은 기존 trades(spot, is_paper) 재사용 — 중복 테이블 없음
```

## 8. 구현 범위

| 단계 | 내용 | 예상 |
|---|---|---|
| MVP | G1/G2 판정 lib + /app/dca 페이지(밸류 존 카드 + 플랜 CRUD + 회차 실행→가상 현물) + 마이그레이션 | 1.5~2일 |
| +α | G3 배분 한도 · G4 스케줄 규율 · 등급 통합 | 0.5~1일 |
| 후속 | 밸류 존 텔레그램 알림(cron 일 1회) · 대시보드 DCA 통계 | 0.5일 |

**전제:** 실LLM 불필요 — 전부 코드 결정론이라 API 키 이슈와 무관하게 구현 가능.

## 9. 명시적 비범위 (MVP에서 안 하는 것)

- 실거래 현물 자동 매수 (Bybit/Binance 실계좌) — 실거래 시스템 로드맵과 함께.
- 온체인/MVRV 등 외부 밸류에이션 지표 — 무료 API 불안정, 3지표 다수결로 시작.
- 매도(익절) 플랜 — 매집만. 매도 규율은 별도 설계.
- AI 코멘터리 — 코드 판정 먼저, AI 보조는 검증 후.

## 10. 검증 결과 (2026-07-13 실행 — ✅ 통과, 채택 형태 확정)

하니스: [`scripts/backtest-dca-valuezone.mjs`](../scripts/backtest-dca-valuezone.mjs) — BTC/ETH 7년 + SOL 4.7년 현물 일봉, 매주 월요일 매수, 미래 정보 없음(전 지표 당일까지 확장 계산).

| 전략 | BTC 평단 | ETH 평단 | SOL 평단 | 최종가치 vs A |
|---|---|---|---|---|
| A 순수 주간 DCA (기준) | — | — | — | — |
| B cheap 온리 (몰아사기) | **−9.4%** | −2.9% | −5.3% | +10.3% / +2.2% / +5.6% |
| C cheap 2x·중립 1x·비쌈 0 | −4.2% | −21.8% | −7.3% | **−6.0%** / +1.6% / +5.0% |
| D cheap 2x·중립 1x·비쌈 0.5x | −1.7% | −8.6% | −1.8% | ≈0 / +0.7% / +1.9% |

**판정:**
- **G2 밸류 존 검증 통과** — 평단 개선이 3심볼 전부 + 거의 모든 연도에서 일관(음수). 참고 강등 불필요.
- **B는 이론상 최강이지만 실전 부적합** — 최장 무매수 BTC 109주 / **ETH 136주(2.6년)**. 2년 넘게 현금 들고 기다리는 규율은 사용자에게 기대 불가.
- **C는 현금 방치 함정** — 상승장에서 expensive 판정에 돈이 놀아 BTC 최종가치 A 대비 −6%.
- **✅ 채택: D(기울인 DCA)를 기본 모드로** — 항상 사되 cheap 2배/중립 1배/비쌈 0.5배. 평단 소폭 개선 + 최종가치 A 동급 이상 + 현금 방치 최소 + 매주 실행되는 규율(습관 유지). B의 "몰아사기"는 **밸류존 진입 텔레그램 알림 + 수동 추가 매수 옵션**으로 제공(강제 아님).
- ⚠️ **SOL 2022 교훈 (나이프 캐칭)**: 폭락 진행 중(260→9)엔 모든 변형이 A보다 나빴음(+8~15%). 대장주(BTC/ETH)에선 완화됨 → **G1 자산 게이트가 필수임을 데이터가 확인**. 후속 검토: 급락 가속 구간(예: 20일 낙폭 급증) cheap 배수 상한 "나이프 가드".
