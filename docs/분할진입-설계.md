# 분할 진입(래더 예약주문) 설계

> 작성: 2026-07-15 · 상태: 설계 확정(구현 전) · 관련: [[위험 예산]] · code-scenario 구조 레벨

## 1. 목적 / 문제

**문제**: 사용자가 분석을 여러 번 하다 보면 같은 코인·방향에 거의 같은 자리로 **대기주문이 실수로 쌓인다**(실측: BTC 롱 4개, 트리거 64,100~64,357). 중복 가드가 체결 포지션만 봐서 pending 중복을 못 막음.

**해결 방향(Victor)**: 겹침을 "막는" 게 아니라, 아예 **의도된 분할 진입(scaling-in)**으로 만든다 — 1·2·3차를 **한 세트 예약주문**으로 내고 **한 포지션**으로 추적. 그러면 중복이 아니라 프로 방식이 된다.

## 2. 현재 상태 (이미 있는 것 / 빠진 것)

**있는 것 (90%)**:
- AI 시나리오 스키마에 `entries` = 1차/2차/3차(tier·price·weight·note) 이미 존재. `synthesize.ts`.
- 규칙 정의됨: weight 합 100, 1차 최대 비중, tier 가격=구조 레벨, **1차~마지막 거리 ≤ 손절폭 절반**, 한도 초과 tier 제거.
- 사이징이 **가중평균 진입가**를 씀(`analysis-result.tsx`, `_actions.ts`).
- 방금 만든 `collectStructuralLevels`(code-scenario) → tier 자리 후보를 코드로 생성 가능(AI 무관).

**빠진 것 (실행 레이어 하나)**:
- 주문 발주 시 **가중평균 1개**로 뭉개 **단일 지정가**만 낸다. → N개 래더 주문으로 내야 함.
- pending/trades가 **주문=거래 1:1** — 여러 tier를 "한 포지션"으로 묶는 개념 없음.

## 3. 핵심 설계 결정

### D1. 주문 그룹핑 = `entry_group_id` (별도 테이블 X)
- `trades` + `pending_limit_orders`에 **`entry_group_id UUID NULL`**, **`entry_tier INT NULL`** 추가.
- N tier = N개 trade(row) + N개 pending_limit_order, **같은 group_id 공유**. 단일주문은 group_id=null.
- "포지션"은 group_id로 **집계한 뷰**(별도 테이블·마이그 최소화). 손절·목표·방향·심볼은 각 row에 이미 있으니 그룹 공유값으로 동일하게 저장.

### D2. 공유 손절/목표 (tier 무관 단일)
- 모든 tier가 **같은 손절·목표**를 가진다(구조 기반 단일 손절, 가장 깊은 tier 아래).
- 가격이 **공유 손절/목표에 닿으면 → 그룹의 체결된 tier 전부 청산 + 미체결 tier 전부 취소**.

### D3. 사이징 = 그룹 전체를 "한 포지션"으로 (핵심)
- 총 수량 = 위험 X%(등급×확신, 위험예산 상한)로 **가중평균 진입가 ↔ 공유 손절** 기준 산정.
- 각 tier 수량 = weight% × 총수량.
- **위험 예산은 그룹당 X% 한 번만** 차지(지금처럼 4배 안 쌓임). money-management가 pending을 tier별로 합산하면 자연히 그룹 총위험이 됨(각 tier |price−stop|×qty 합 = 정확한 총위험).

### D4. 부분 체결 처리
- 각 tier는 독립 지정가 → 가격 닿으면 **filler가 개별 체결**(기존 로직 재사용).
- 포지션 = **체결된 tier 합**. 평균진입 = 체결분 실체결가 가중평균. 미체결 tier는 대기.
- 1차만 체결/1·2차 체결 → 그만큼만 노출(계획보다 **적은 위험** = 보수적, OK). 되돌림이 깊게 안 오면 under-risk.

### D5. 만료 / 취소
- tier별 만료(또는 그룹 공유 만료). 2·3차 미도달 시 만료 → 체결분은 포지션 유지.
- 그룹 취소 = 미체결 tier 전부 취소. 체결분은 단독 포지션으로 남김(또는 청산 제안).
- 1차 체결 + 2·3차 대기 중 손절 닿음 → 1차 청산 + 2·3차 취소(D2).

### D6. 폴백 tier 생성 (AI 무관)
- `code-scenario`가 단일 진입 대신 **구조 레벨로 1~3 tier 생성**: 1차=가장 가까운 지지, 2차=다음, 3차=최후(전부 [MIN_GAP, 손절폭 절반] 안). weight 40/35/25. 구조 레벨 1개뿐이면 단일 tier(weight 100).

### D7. v1 범위 제한
- **되돌림(LIMIT) 래더만**. 돌파(STOP)·즉시(market)는 단일 주문 유지(래더 대상 아님).
- tier 각각 **수수료 가드·트리거 가드** 통과해야 함(손절에 너무 가까운 깊은 tier는 제거).

## 4. 데이터 모델 (마이그레이션 1개)

```sql
ALTER TABLE trades
  ADD COLUMN IF NOT EXISTS entry_group_id UUID,
  ADD COLUMN IF NOT EXISTS entry_tier INT,
  ADD COLUMN IF NOT EXISTS entry_weight NUMERIC;
ALTER TABLE pending_limit_orders
  ADD COLUMN IF NOT EXISTS entry_group_id UUID,
  ADD COLUMN IF NOT EXISTS entry_tier INT;
CREATE INDEX IF NOT EXISTS trades_entry_group_idx ON trades (entry_group_id) WHERE entry_group_id IS NOT NULL;
```
(별도 `entry_groups` 테이블은 v2에서 필요시. v1은 group_id 집계로 충분.)

## 5. 실행 흐름

1. **발주**(서버액션): 시나리오 tiers 받음 → group_id 1개 생성 → tier마다 trade(pending) + pending_limit_order 삽입(공유 손절/목표, tier별 limit_price/weight/qty). 총수량은 위험예산 상한 안에서 가중평균 기준 산정.
2. **체결**(filler): tier별 개별 체결. 체결 시 그 trade order_status=filled.
3. **정산**(resolve-trades): **그룹 인지** — 공유 손절/목표가 현재가에 닿으면 그룹의 체결 tier 전부 청산 + 미체결 tier 취소. 평균진입 기준 result_r.
4. **집계 뷰**: group_id로 묶어 평균진입·총수량·미실현 손익·체결 진행(예: 2/3 tier 체결) 표시.

## 6. UI

- 분석 결과/거래폼: "**분할 진입 (3단)**" 카드 — tier별 가격·비중·거리, 가중평균 진입가, 공유 손절/목표, 총 위험/노출.
- **한 번에 예약** 버튼 → 그룹 발주.
- 거래소 화면 대기주문: 같은 group은 **묶어서 표시**("BTC 롱 분할 1/3 · 2/3 · 3/3"), 그룹 단위 취소.
- 진행 포지션: "분할 진입 2/3 체결 · 평균 $X" 형태.

## 7. 단계별 구현

- **1단계 (MVP)**: 마이그 + 발주(그룹 N주문) + filler 그룹 무관 개별체결(이미 됨) + resolve 그룹 청산 + 기본 UI(분할 카드·묶음 표시). 부분체결은 "체결분만 포지션".
- **2단계**: 부분체결 평균가·위험 실시간 재계산 정밀화 + 그룹 취소/만료 UX + 위험예산 그룹 집계 검증.
- **폴백 tier**: code-scenario 구조레벨 tier 생성(1단계와 병행 가능).

## 8. 엣지 케이스

- tier 가격이 서로 너무 가까움 → 병합(단일 tier).
- 깊은 tier가 손절에 너무 가까워 수수료 가드 위반 → 그 tier 제거.
- 1차가 현재가 근처(즉시) → v1은 래더 제외(단일). v2에서 "즉시 1차 + 되돌림 추가" 허용 검토.
- 그룹 일부 체결 후 사용자가 수동 청산 → 미체결 tier 자동 취소.
- 위험예산이 부족 → tier 수/총수량 축소(가장 깊은 tier부터 제거).

## 9. 확정 결정 (Victor 2026-07-15)

- **tier 수** = 시나리오/구조가 주는 대로 (1~3, **최대 3**). 고정 개수 아님.
- **weight** = v1은 **시나리오 값(기본 40/35/25) 고정**. 사용자 조절은 v2.
- **부분체결 손절** = **체결분만 손절**(보수적) + **미체결 tier 자동취소**.
